================================================

Content: Cangjie 5th Generation (倉頡第五代) code -> single Han character mapping.
Format:  Converted to "<code>\t<char>" per line. The upstream frequency column
         (a trailing "\t<freq>") was dropped; rows are otherwise verbatim.

Yahoo! KeyKey 三代 tables (倉頡第三代 / 速成)
============================================

Files:   Resources/cangjie-yahoo.txt  (Cangjie 3rd Generation, 倉頡第三代)
         Resources/simplex-yahoo.txt  (Simplex / 速成, quick codes)
Content: <code>\t<char> mapping used by the "三代倉頡（Yahoo KeyKey 相容）" mode.
         The line order is preserved verbatim — it is the original Yahoo! KeyKey
         candidate order, which that mode surfaces directly.
Format:  Extracted from the source .cin %chardef sections; only rows whose code is
         a pure a–z sequence are kept; the trailing frequency column is dropped and
         CRLF line endings are normalised to LF. Reproduce with:
           curl -s <raw-url> | tr -d '\r' \
             | awk '/%chardef begin/{f=1;next}/%chardef end/{f=0}f{n=split($0,a,/[ \t]+/); \
                    if(a[1] ~ /^[a-z]+$/ && a[2]!="") print a[1]"\t"a[2]}'
